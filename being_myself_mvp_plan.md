## 1. 서비스 개요

**서비스명**: Being Myself  
**슬로건**: 하루 15분, 나에게 집중하는 시간  
**한 줄 정의**: AI 페이스메이커와의 대화를 통해 내가 좋아하고 잘하고 가치있게 여기는 것을 발견하고, 소명의 실마리를 찾아가는 서비스

Being Myself는 모비니티 온톨로지의 소명 여정 5단계 중 **2단계(발견, Discovery)** 를 책임지는 핵심 도구다. 소명을 발견하는 과정은 단번에 이루어지지 않는다. 매일 짧은 대화를 반복하면서 파편화된 자기 이해가 연결되고, 어느 순간 "아, 이거구나"라는 유레카의 순간이 찾아온다. Being Myself는 그 과정을 AI가 함께 걸어가는 방식으로 구현한다.

## 2. 서비스 목적 및 배경

### 2.1 왜 Being Myself인가

기존 Being Myself 레거시 버전에서 사람 코치(페이스메이커)가 수행하던 역할이 있었다. 표면적인 답변에서 더 깊은 본질을 뽑아내고, 그것을 논리적으로 정리해주는 역할이다. 이 경험은 사용자에게 강한 감동을 주었으나 사람 코치가 필요하다는 확장성의 한계가 있었다.

MVP는 이 역할을 AI로 대체하는 첫 번째 시도다. 사람이 직접 수행하며 전달하던 정서적 감동과 깊이 있는 통찰을 AI와 엔지니어링 기술로 구현하는 것이 목표다.

### 2.2 온톨로지와의 연결

Being Myself는 모비니티 온톨로지의 소명 내부 구조를 기반으로 설계된다.

```
정체성 (Identity) — 존재의 기반. "나는 누구인가"
      ↓
강점 (Strength) — 존재의 무기. "나는 무엇을 잘하는가"
      ↓
소명 (Calling) — 존재의 이유. "정체성과 강점이 세상의 필요와 만나는 지점"
```

셀프인터뷰의 세 파트(좋아하는 것 / 잘하는 것 / 가치있는 것)는 각각 이 구조의 입력 데이터가 된다. AI 페이스메이커는 이 데이터를 기반으로 더 깊이 파고들며 소명의 실마리를 함께 찾아간다.

### 2.3 MVP의 목적

이번 MVP는 두 가지 목적을 동시에 달성한다.

**1) 박람회 데모**: 기독교 리더십 박람회(교회 목사, 교육 담당자 대상)에서 Being Myself 서비스를 시연한다. 방문자가 직접 경험하고 소명 발견의 첫 실마리를 느낄 수 있도록 한다.

**2) 내부 구성원 서비스**: 기존에 셀프인터뷰를 완료한 모비니티 구성원들이 실제로 사용할 수 있는 AI 페이스메이킹 루틴을 제공한다.

## 3. 대상 사용자

### 3.1 내부 구성원 (로그인 사용자)

- 기존 셀프인터뷰(좋아하는 것 / 잘하는 것 / 가치있는 것) 데이터 보유
- 매일 5분, AI와의 대화를 통해 소명을 더 깊이 발견해가는 지속적 루틴
- 대화가 누적될수록 보고서가 점점 선명해지는 경험

### 3.2 박람회 방문자 (게스트 사용자)

- 기독교 리더십 박람회 방문자 (목사, 교육 담당자, 리더 등)
- 셀프인터뷰 데이터 없음 → 미리 세팅된 페르소나 데이터 기반으로 데모 진행
- 담당자가 페르소나를 선택해주고 방문자는 바로 대화 경험 시작
- 미리 만들어진 선택지 중 선택 → 보고서 확인 → 세션 종료 (다음 방문자를 위해 리셋)

### 3.3 박람회 페르소나 3종

| 페르소나 | 대상 | 특징 |
|---|---|---|
| 사역자 | 담임목사, 부교역자 | 말씀 중심, 공동체 지향, 장기적 시각 |
| 직장인 | 신앙을 가진 직장인 | 일과 신앙의 통합, 개인 역량 중심 |
| 중고등학생 | 청소년 | 방향 탐색 중, 관계와 재미 중심 |

## 4. 핵심 경험 정의

### 4.1 Being Myself가 주는 경험

사용자가 Being Myself를 사용하면서 느껴야 할 핵심 경험은 다음과 같다.

> "대화를 하다 보니 어느새 내 이야기가 정리되어 있었다."

이를 위해 AI 페이스메이커는 세 가지 원칙을 지킨다.

**원칙 1. 의도가 느껴지지 않게**  
분석하는 느낌이 아니라 대화하는 느낌. 질문의 목적이 드러나지 않도록 자연스럽게 흐른다.

**원칙 2. 낙인 없이**  
"당신은 이런 사람입니다"라고 결론 내리지 않는다. "이 부분들이 서로 연결되는 것 같아요"라고 연결선만 보여준다. 판단은 사용자의 몫으로 남긴다.

**원칙 3. 허들을 낮게**  
5분 이내, 단어 하나나 짧은 문장으로도 충분하다. 텍스트와 음성 모두 지원하여 답변 방식을 사용자가 선택한다.

### 4.2 페이스메이커의 역할

페이스메이커는 나의 생각을 열어주는 '질문'을 던져주는 존재다. 단순히 답변을 듣는 것이 아니라, 표면적인 답변 아래에 있는 본질을 함께 찾아간다.

```
표면 답변 → 본질 추출 → 가치 연결 → 존재 확인
```

예시:
- "교육이 좋아요" (표면)
- "교육의 어떤 순간에 가장 살아있다고 느끼나요?" (본질 추출)
- "그 순간이 당신이 소중히 여기는 것과 어떻게 연결되나요?" (가치 연결)
- "이런 순간들이 모여 당신을 어떤 사람으로 만들고 있나요?" (존재 확인)

## 5. 사용자 플로우

### 5.1 내부 구성원 플로우

```
앱 진입
    ↓
로그인 (Google 계정)
    ↓
홈 화면 — 오늘의 질문 확인
    ↓
AI 페이스메이커 대화 시작
    ├── 텍스트 입력
    └── 음성 입력 (STT → 텍스트 변환)
    ↓
질문 1 → 답변
    ↓
질문 2 → 답변 (전날/이전 답변 기반 심화)
    ↓
오늘의 대화 요약 확인
    ↓
보고서 확인 (누적 업데이트)
    ↓
세션 종료
```

**반복 구조 (나선형)**  
매일 대화가 누적되면서 질문이 점점 깊어진다.
- Day 1: 기존 인터뷰 데이터 기반 씨앗 질문
- Day 2~: 전날 답변에서 AI가 "더 파고들 지점"을 골라 다음 질문의 씨앗으로 사용
- 매 순환마다 같은 자리가 아닌 더 깊은 층위에서 시작

### 5.2 박람회 게스트 플로우

```
담당자가 페르소나 선택
(사역자 / 직장인 / 중고등학생)
    ↓
방문자에게 기기 전달
    ↓
게스트 화면 진입
    ↓
페르소나 기반 AI 웰컴 메시지 확인
    ↓
질문 1 확인 + 선택지 3개 중 선택
    ↓
질문 2 확인 + 선택지 3개 중 선택
    ↓
보고서 생성 (미리 만들어진 보고서 불러오기)
    ↓
보고서 확인
    ↓
세션 종료 → 다음 방문자를 위해 초기화
```

**게스트 세션의 특징**
- 답변이 다음 세션에 반영되지 않음
- 네트워크 의존 최소화 (보고서는 미리 생성된 것을 불러옴)
- 담당자가 언제든 페르소나 변경 및 초기화 가능

## 6. 화면 설계

### 6.1 전체 화면 목록

| 화면 ID | 화면명 | 대상 |
|---|---|---|
| S-00 | 스플래시 / 진입 화면 | 공통 |
| S-01 | 로그인 화면 | 내부 구성원 |
| S-02 | 담당자 페르소나 선택 화면 | 박람회 담당자 |
| S-03 | 게스트 진입 화면 | 박람회 방문자 |
| S-04 | 홈 화면 | 내부 구성원 |
| S-05 | 대화 화면 (AI 페이스메이킹) | 공통 |
| S-06 | 선택지 화면 (박람회 전용) | 박람회 방문자 |
| S-07 | 오늘의 대화 요약 | 내부 구성원 |
| S-08 | 보고서 화면 | 공통 |
| S-09 | 보고서 상세 — 파트별 | 공통 |

### 6.2 S-00 스플래시 / 진입 화면

**목적**: 서비스의 첫인상. "나에게 집중하는 시간"이라는 분위기를 전달한다.

**구성 요소**
- Being Myself 로고
- 슬로건: "하루 15분, 나에게 집중하는 시간"
- 로그인 버튼 (Google 계정으로 로그인)
- 게스트로 시작하기 버튼 (박람회 방문자용)

**디자인 노트**
- 다크 배경 + 보라 그라데이션 포인트
- 레거시 일러스트 캐릭터 활용

---

### 6.3 S-01 로그인 화면

**목적**: 기존 구성원 인증

**구성 요소**
- Google 계정으로 로그인 버튼
- 로그인 시 기존 인터뷰 데이터 자동 불러오기

---

### 6.4 S-02 담당자 페르소나 선택 화면

**목적**: 박람회 담당자가 방문자에게 맞는 페르소나를 세팅한다. 방문자는 이 화면을 보지 않는다.

**구성 요소**
- 페르소나 3개 카드
  - 사역자 (최병호 목사, 45세)
  - 직장인 (정다은, 29세)
  - 중고등학생 (이준혁, 15세)
- 각 카드: 이름, 나이, 한 줄 설명, 선택 버튼
- 선택 후 게스트 화면(S-03)으로 이동
- 초기화 버튼 (세션 종료 후 다시 시작할 때)

**인터랙션**
- 페르소나 선택 → 확인 팝업 ("이 페르소나로 시작할까요?") → 확인 시 게스트 화면 진입

---

### 6.5 S-03 게스트 진입 화면

**목적**: 방문자가 처음 보는 화면. 서비스의 목적과 경험을 간단히 안내한다.

**구성 요소**
- 환영 메시지: "안녕하세요, [페르소나명]님"
- 서비스 소개 한 줄: "오늘 단 두 가지 질문으로 나의 소명을 발견해보세요"
- AI 웰컴 메시지 (페르소나별 맞춤)
- 시작하기 버튼

**AI 웰컴 메시지 예시 (사역자)**
> "최병호 목사님, 반갑습니다. 설교 준비부터 심방까지 바쁜 사역의 시간 중에, 오늘은 잠깐 나 자신에게 집중하는 시간을 가져보시겠어요?"

---

### 6.6 S-04 홈 화면 (내부 구성원)

**목적**: 매일 접속하는 메인 화면. 오늘의 질문과 누적 현황을 보여준다.

**구성 요소**
- 상단: 사용자 이름 + 날짜
- 오늘의 질문 카드
  - AI가 준비한 오늘의 첫 번째 질문 미리보기
  - "대화 시작하기" 버튼
- 소명 지도 현황
  - 정체성 / 강점 / 소명 3층 구조 진행도 (선명해지는 정도)
  - "지금까지 N번의 대화를 나눴어요"
- 보고서 바로가기

---

### 6.7 S-05 대화 화면 (AI 페이스메이킹)

**목적**: Being Myself의 핵심 화면. AI와 실제 대화가 이루어지는 공간.

**구성 요소**
- 상단: 진행 표시 (질문 1/2)
- AI 메시지 영역
  - AI 아바타 (Being Myself 캐릭터)
  - AI 질문 텍스트
  - 맥락 연결 문구 (전날 답변과 연결될 때: "지난번에 말씀하신 ~와 연결해서 여쭤볼게요")
- 사용자 입력 영역
  - 텍스트 입력창 ("짧게 한 문장이어도 좋아요")
  - 마이크 버튼 (음성 입력)
  - 전송 버튼
- 하단: "건너뛰기" (선택 사항)

**내부 구성원 대화 흐름**
1. AI 질문 표시
2. 사용자 텍스트 or 음성 답변
3. AI 짧은 리액션 ("그렇군요", "흥미롭네요" 등)
4. 다음 질문으로 이어지거나 마무리

---

### 6.8 S-06 선택지 화면 (박람회 전용)

**목적**: 박람회 방문자가 타이핑 없이 선택만으로 대화할 수 있게 한다.

**구성 요소**
- AI 질문 텍스트
- 선택지 카드 3개
  - 각 카드: 짧은 답변 텍스트 (1~2문장)
  - 탭하면 선택됨
- 선택 후 "다음" 버튼 활성화

**인터랙션**
- 선택지 탭 → 카드 하이라이트
- "다음" 버튼 탭 → 선택 확정 + 다음 질문으로 이동
- 두 번째 질문 동일 방식 진행
- 두 번 선택 완료 → 보고서 생성 로딩 → 보고서 화면(S-08)으로 이동

---

### 6.9 S-07 오늘의 대화 요약 (내부 구성원)

**목적**: 대화가 끝난 후 오늘 나눈 이야기를 간단히 정리해준다.

**구성 요소**
- "오늘 [이름]님과 나눈 이야기에요" 헤더
- 오늘 질문 + 답변 요약 (2개)
- AI의 오늘 인사이트 한 줄
  - 예: "오늘 말씀하신 '완성되는 순간보다 몰입하는 과정'이라는 표현이 인상 깊었어요."
- 보고서 업데이트 알림: "소명 지도가 조금 더 선명해졌어요"
- 보고서 보기 버튼
- 홈으로 버튼

### 6.10 S-08 보고서 화면

**목적**: 지금까지의 대화를 통해 발견된 나의 모습을 정리해서 보여준다. "진행 중인 지도" 개념.

**구성 요소**
- 상단 헤더
  - "내가 진정 하고 싶은 일은 [한 줄 요약]이라고 합니다."
  - 부제: "다른 사람이 아닌, 오직 나만이 느끼고 알 수 있는 일이죠."
- 파트 탭 (또는 스크롤)
  - Part 1. 좋아하는 것 I Love It
  - Part 2. 잘하는 것 I Am Great at It
  - Part 3. 가치있는 것 I Feel Valuable About It
- 각 파트: 질문 + 항목들 + 💡 인사이트 요약 (핵심 키워드 포함)
- 하단: 공유하기 버튼 (선택)

## 7. 콘텐츠 구조

### 7.1 내부 구성원 데이터 구조

```
User
├── profile (이름, 이메일)
├── interview_data
│   ├── likes (좋아하는 것 답변들)
│   ├── strengths (잘하는 것 답변들)
│   └── values (가치있는 것 답변들)
├── conversations (대화 기록)
│   ├── date
│   ├── questions (AI 질문들)
│   └── answers (사용자 답변들)
└── report (누적 보고서)
    ├── one_line_vision
    ├── part1_likes
    ├── part2_strengths
    └── part3_values
```

### 7.2 박람회 페르소나 데이터 구조

```
Persona
├── id (사역자 / 직장인 / 중고등학생)
├── profile (이름, 나이, 직업)
├── base_interview_data
│   ├── likes
│   ├── strengths
│   └── values
├── questions
│   ├── question_1
│   │   ├── text
│   │   └── choices (3개)
│   └── question_2
│       ├── text
│       └── choices (3개)
└── reports
    ├── report_A (선택 조합 A)
    └── report_B (선택 조합 B)
```

### 7.3 박람회 선택지 → 보고서 매핑

선택 조합을 단순화하기 위해 선택지를 두 방향으로 분류한다.

- 선택지 A, B → 방향 1 보고서
- 선택지 C → 방향 2 보고서

페르소나별로 보고서 2종 (방향 1 / 방향 2)을 미리 제작한다.

총 필요 보고서: 3 페르소나 × 2 방향 = **6종**

## 8. AI 페이스메이커 설계

### 8.1 질문 생성 로직 (내부 구성원)

**Day 1 — 씨앗 질문**
- 기존 인터뷰 데이터(likes, strengths, values)에서 가장 핵심적인 키워드를 추출
- 아직 충분히 탐색되지 않은 영역을 우선 선택
- 공통 질문 형식으로 시작

**Day N — 심화 질문**
- 전날 사용자 답변을 분석
- AI가 "더 파고들 지점"을 식별 (동사, 감정어, 반복 키워드)
- 해당 지점을 새로운 질문의 씨앗으로 사용
- 맥락 연결 문구 추가: "지난번에 말씀하신 ~"

**질문 층위 모델**
```
1단계: 현상 기록 — "무엇을 했나요?"
2단계: 본질 추출 — "그 안의 어떤 포인트가 당신을 움직였나요?"
3단계: 가치 연결 — "그 움직임은 당신이 소중히 여기는 것과 어떻게 닿아 있나요?"
4단계: 존재 확인 — "이런 순간들이 모여 당신을 어떤 사람으로 만들고 있나요?"
```

### 8.2 질문 설계 원칙

**의도 숨기기**  
"당신의 강점이 무엇인지 알아보겠습니다" 같은 직접적 표현 금지. 자연스러운 대화처럼 느껴지게 한다.

**부담 최소화**  
- "짧게 한 문장이어도 좋아요"
- "떠오르는 단어만 적어주셔도 됩니다"
- 건너뛰기 옵션 제공

**맞장구와 미러링**  
사용자 답변을 받은 후 AI가 짧게 반응한다.
- "그렇군요, 흥미롭네요"
- "그 표현이 인상 깊어요"
- 사용자의 언어를 그대로 반영해서 돌려준다

**조건부 강점 파악**  
"그 능력이 언제나 발휘되나요, 아니면 특정한 상황이 갖춰져야 하나요?"

### 8.3 AI 리액션 설계

사용자가 답변했을 때 AI가 보내는 짧은 반응. 감동 포인트.

| 상황 | AI 리액션 예시 |
|---|---|
| 깊은 답변이 나왔을 때 | "그 표현, 참 정확하네요." |
| 결핍에서 온 답변일 때 | "그 경험이 오히려 지금의 [강점]을 만든 것 같아요." |
| 짧게 답변했을 때 | "충분해요. 그 한 마디 안에 많은 게 담겨있어요." |
| 모르겠다고 할 때 | "모르는 게 당연해요. 그래서 우리가 함께 찾는 거예요." |

### 8.4 박람회 AI 설계

박람회에서는 실시간 AI 생성 없이 미리 설계된 흐름으로 진행한다.

- 웰컴 메시지: 페르소나별 맞춤 텍스트 (미리 작성)
- 질문: 페르소나별 고정 질문 2개
- 선택지: 각 질문당 3개의 미리 작성된 답변 옵션
- 리액션: 선택에 따른 미리 작성된 AI 반응 텍스트
- 보고서: 선택 조합에 따라 미리 생성된 보고서 불러오기

### 9.2 보고서 생성 원칙

**낙인 없이**  
"당신은 이런 사람입니다" → "이런 순간들이 서로 연결되어 있어요"

**진행형으로**  
완성된 정의가 아닌, 발견해가는 과정임을 강조한다.  
"지금 이 시점에서 [이름]님이 발견한 모습이에요. 앞으로 더 깊어질 거예요."

**사용자 언어 그대로**  
AI가 임의로 해석하지 않고, 사용자가 직접 사용한 표현을 최대한 그대로 반영한다.

**인사이트 요약의 톤**  
- 따뜻하고 중립적
- 결론 내리지 않고 연결선만 제시
- 핵심 키워드는 사용자가 직접 사용한 단어에서 추출

### 9.3 내부 구성원 보고서 업데이트 방식

- 대화가 누적될수록 보고서가 점점 선명해진다
- 초기: 기존 인터뷰 데이터 기반의 1차 보고서
- 대화 5회 이후: AI가 새로운 패턴 발견 시 항목 추가 or 수정 제안
- 사용자가 직접 수정 가능 ("이 부분은 좀 다르게 표현하고 싶어요")

## 11. 기술 스펙

### 11.1 플랫폼

- 웹앱 (모바일 우선 반응형)
- 박람회: 태블릿/모바일 기기에서 브라우저로 실행
- 내부 구성원: 모바일 + 데스크톱 모두 지원

### 11.2 인증

- 내부 구성원: Google OAuth 2.0
- 박람회 게스트: 인증 없음 (담당자 화면에서 페르소나 선택 후 게스트 세션 시작)

### 11.3 음성 입력

- 사용자가 텍스트 / 음성 중 선택
- 음성: Web Speech API (브라우저 내장) 또는 Whisper API
- 음성 → STT → 텍스트로 변환 후 동일하게 처리
- 음성 입력 중 실시간 파형 표시 (레거시 스타일)

### 11.4 AI 연동 (내부 구성원)

- Anthropic Claude API 사용
- 사용자 인터뷰 데이터 + 이전 대화 기록을 컨텍스트로 제공
- 질문 생성 → 답변 수신 → 보고서 업데이트 순서로 처리
- 응답 스트리밍으로 타이핑 효과 구현

### 11.5 AI 연동 (박람회)

- 실시간 API 호출 없음
- 모든 질문, 선택지, 리액션, 보고서 미리 제작 후 DB에 저장
- 선택 조합에 따라 해당 데이터를 불러오는 방식
- 네트워크 불안정 환경에서도 안정적으로 동작

### 11.6 데이터베이스

**내부 구성원**
- Users: 사용자 프로필, 인터뷰 데이터
- Conversations: 대화 기록 (날짜, 질문, 답변)
- Reports: 누적 보고서 데이터

**박람회**
- Personas: 페르소나 기본 데이터, 질문, 선택지
- Demo_Reports: 미리 생성된 보고서 (페르소나 × 방향)
- Guest_Sessions: 임시 세션 데이터 (세션 종료 시 삭제)

### 11.7 개발 우선순위

**Phase 1 (박람회 데모)**
1. 게스트 플로우 전체 (S-02 ~ S-08)
2. 3개 페르소나 콘텐츠 입력
3. 6종 보고서 제작 및 연동
4. 레거시 디자인 시스템 구현

**Phase 2 (내부 구성원)**
1. 로그인 / 인증
2. 홈 화면
3. AI 페이스메이킹 대화 화면
4. 음성 입력
5. 보고서 누적 업데이트

- **소명 지도 시각화**: 정체성 → 강점 → 소명 3층 구조가 선명해지는 시각적 표현

### 13.2 중기

- **Being Us 연동**: 개인 소명이 팀 소명으로 연결되는 플로우
- **CAI 연동**: Calling Alignment Index와 연결하여 소명-업무 정렬도 측정
- **보고서 공유**: 동료와 소명을 나누는 기능

### 13.3 장기

- **소명 여정 전체 커버**: Being Myself(발견) → CAI(정렬) → MBS(실현)으로 이어지는 통합 플랫폼
- **외부 사용자 오픈**: 모비니티 고객사 구성원 대상 서비스 확장
- **문화 성숙도 연동**: 개인의 실현 데이터가 조직 문화 지표로 연결
