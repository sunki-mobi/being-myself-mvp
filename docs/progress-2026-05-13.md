# Being Myself — 박람회 이후 진행 (2026-05-13)

박람회(5/12) 다음 날 하루 동안 **단일 사용자 시연 → 다중 사용자 실 서비스**로 첫 전환. 회원가입·답변 누적·자기 baseline 생성·누적 보고서까지 풀 흐름이 dev·prod 모두에서 동작합니다.

> **한 줄 요약**
> 이제 누구나 본인 계정으로 가입하고, 자기 셀프인터뷰를 끝내고, 매일 답변이 자동으로 정리되는 보고서를 본인의 데이터로 받을 수 있어요.

---

## 1. 회사 Supabase·Gemini로 마이그레이션 (인프라)

- 개인 계정으로 만들었던 Supabase·Gemini API 키를 회사 organization·API 키로 교체
- dev(노트북)·prod(Vercel) 양쪽 환경변수 갱신
- 옛 프로젝트는 비활성·삭제 예정 (실 사용자 데이터 없음 — 본인 테스트 데이터만)

---

## 2. 회원가입·로그인 시스템 (Phase 1)

- **Supabase Auth** (이메일+비밀번호) 도입. Google OAuth는 후속 단계.
- **DB 스키마·테이블**: `profiles`, `user_settings` + RLS(Row-Level Security)
- Auth에 새 사용자가 가입하면 자동으로 본인 row 생성 (DB 트리거)
- `/me/*` 모든 경로에 비로그인 가드 (proxy 미들웨어)
- `/auth/sign-in`, `/auth/sign-up`, `/auth/callback`, `/auth/sign-out` 라우팅
- Next.js 16 `middleware` → `proxy` 명명 변경 반영

---

## 3. 매일 답변 누적 (Phase 2)

- **DB 스키마**: `conversation`, `qa_pair`, `answer_card` + RLS
- 사용자가 답변할 때마다:
  - LocalStorage (즉시 UX 반영)
  - Supabase DB (누적 저장)
- 게스트 데모(`/demo`)는 그대로 LocalStorage만 — 익명 게스트 시연 유지

---

## 4. baseline 셀프인터뷰 — 본인 보고서 생성 (Phase 3)

신규 사용자가 자기 baseline을 만드는 **두 갈래** 경로:

### (a) 음성 셀프인터뷰 (Phase 3a + 3b)
- **6 step 흐름**: 3개 질문 블록 × (객관식 → 음성 답변)
- **Web Speech API**로 한국어 실시간 음성 인식 — 말하는 동안 화면에 글이 떠오름. Gemini API 호출 0회로 무료.
- **이탈 후 이어서 가능** — 답변 단계마다 DB에 즉시 저장, 다음 진입 시 정확히 그 자리에서
- 마지막 단계에서 **LLM이 모든 답변을 BaselineReport JSON으로 합성**

### (b) 기존 자료 가져오기 — Import (Phase 3c)
- 이미 셀프인터뷰 자료를 갖고 있는 사람용 (PDF·메모·노트 등)
- 텍스트를 통째로 paste → LLM이 자동으로 3 Part(가치/좋아하는/잘하는)로 분류·정리
- 사용자가 검토·수정 화면에서 직접 편집 후 저장

### (c) 보고서 DB wire-up (Phase 3d)
- `/me/report`·`/me/report/full` 페이지가 본인 baseline을 읽어 표시
- 기존 박람회용 SKPAN 하드코딩 제거 → 실 사용자별 본인 데이터
- 인터뷰·Import 둘 다 끝나면 `/me` 화면이 자동으로 baseline 보고서 카드로 분기

---

## 5. 4단계 누적 보고서 — "지금까지의 모습" (Phase 4a)

- 답변이 쌓일수록 진화하는 본인의 종합 보고서
- **4단계 — 현상(일상) · 본질(움직이는 결) · 가치(소중한 것) · 존재(되고 싶은 모습)**
- baseline 만든 직후부터 즉시 LLM이 best-effort 합성 → **Day 1부터 와우 모먼트 즉시 제공**
- 임계점 기다리지 않고 항상 제공. 답변 누적에 따라 점진적으로 내용 진화 (다음 페이즈에서 재합성 트리거 추가 예정)

---

## 6. prod 운영 셋업

- Vercel 환경변수 3개 등록 (Supabase URL·key + Gemini API key)
- prod 도메인 `https://being-myself-mvp.vercel.app` 에서 풀 흐름 동작
- env 누락 시 우아한 fallback (`/demo`는 항상 살아남도록 — 박람회 직후 prod 다운 방어)
